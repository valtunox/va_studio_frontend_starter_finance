# VA Studio Frontend Starter

Vite + React starter with 23 pre-built templates for rapid app development. Powers the user-facing frontend of projects created in VA Studio.

This repo is embedded by **va_studio_ai_builder_backend** (Go, port 8743) to provide instant UI for user-created projects.

## Role in VA Studio Architecture

```
  va_studio_ai_builder_frontend (Next.js :3000)
    |-- Studio UI, auth, project management
    |-- Cloud IDE with live preview
            |
            v
  va_studio_ai_builder_backend (Go/Gin :8743)
    |-- Orchestrates project lifecycle
    |-- Runs AI code modifications on templates
    |-- Serves live preview
            |                          |
            v                          v
  va_studio_backend_starter        va_studio_frontend_starter
  (FastAPI :5112)                  (Vite+React :3008) <-- THIS REPO
    |-- Project API                  |-- Project UI
    |-- Auth, CRUD, billing          |-- 23 templates
            |                          |
            +----------+---------------+
                       |
                    User's App
```

### How It Fits

1. User picks a template in VA Studio (e.g., "E-commerce store")
2. The AI builder backend copies this starter and configures the selected template
3. AI modifies components based on user prompts ("add a dark mode toggle", "change the pricing page")
4. This starter runs as the project's frontend, consuming the backend starter API
5. User deploys the final app via Docker

### Integration Points

| System | Connection |
|--------|------------|
| **va_studio_ai_builder_backend** | Orchestrator -- scaffolds, modifies, and serves this as live preview |
| **va_studio_backend_starter** | Paired backend -- this frontend calls its API at `http://localhost:5112` |
| **va_studio_ai_builder_frontend** | Studio UI -- embeds this in an iframe for live preview |

### Switching Templates

Edit `src/App.jsx` to re-export the desired template:

```js
export { default } from '../templates/portfolio/App.jsx'
export { default } from '../templates/saas/App.jsx'
export { default } from '../templates/ecommerce/App.jsx'
export { default } from '../templates/crm/App.jsx'
export { default } from '../templates/erp/App.jsx'
```

Or use the built-in preview system: navigate to `/preview/:templateId` to view any template.

## Auth Components

Ready-to-use auth UI that connects to the backend starter's JWT endpoints:

```jsx
import { LoginForm, RegisterForm, OAuthButtons, AuthModals } from '@/components/auth'

// Inline form
<LoginForm onSubmit={handleLogin} onOAuth={handleOAuth} onForgotPassword={() => setMode('reset')} />

// Modal with login/register/reset
<AuthModals
  open={showAuth}
  onOpenChange={setShowAuth}
  mode={mode}
  onModeChange={setMode}
  onLogin={handleLogin}
  onRegister={handleRegister}
  onOAuth={handleOAuth}
/>
```

## Backend Connectivity

Built-in real-time backend status monitoring:

- Banner at top of page shows connection status (online/offline/degraded)
- Latency display in milliseconds
- Auto-polling: 30s when online, 10s when offline
- Retry button for manual refresh
- Context API: `useBackendStatus()` hook available globally

## Quick Start

```bash
npm install
npm run dev
# App runs at http://localhost:3008
# API proxied to http://localhost:5112
```

### Using Docker

```bash
docker-compose up -d
# Frontend: http://localhost:3008
# Nginx proxy: ports 80/443
```

## Component Index

See `templates/COMPONENT_INDEX.md` for AI context and component discovery. This file maps every component across all 23 templates, enabling the AI builder to find and modify the right files.

## Deployment

### Docker (Production)

```bash
# Build
docker build -t va-studio-frontend:latest \
  --build-arg VITE_API_URL=https://api.yourdomain.com \
  --build-arg VITE_APP_NAME="My App" .

# Run
docker run -p 3008:3008 va-studio-frontend:latest
```

### With Nginx SSL

```bash
docker-compose -f docker-compose.deploy.yml up -d
# Serves on ports 80/443 with SSL termination
```

Resource limits (production):
- Frontend: 1 CPU, 1GB RAM
- Nginx: 0.5 CPU, 512MB RAM
